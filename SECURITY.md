# Security Policy

## Supported Versions

Security fixes are applied to the latest released version of this SDK.
If you are running an older version, upgrade to the latest release before
reporting a vulnerability unless the issue is specific to a historical release.

## Reporting a Vulnerability

Please do **not** open public issues for security vulnerabilities.

Use one of the following channels:

1. Email: security@geoidenti.com
2. GitHub Security Advisory (preferred for private collaboration)

Include the following details in your report:

- Affected package version
- Impacted component(s) and file paths
- Clear reproduction steps or proof of concept
- Potential impact and attack preconditions
- Any suggested mitigations

## What to Expect

- Initial acknowledgment target: within 3 business days
- Triage and severity assessment after acknowledgment
- Coordinated disclosure once a fix is available

## Disclosure Guidelines

- Keep vulnerability details private until maintainers confirm disclosure timing.
- Avoid publishing exploit details or proof-of-concept code before a fix ships.
